# Git Tool 5.1.4

一个面向 Git 初学者的 **Git 图形化自动工具**。  
它把常用 Git / GitHub 命令做成按钮，用户不用反复输入命令，也可以完成初始化仓库、提交、推送、拉取、查看日志、恢复文件、生成 `.gitignore` 等操作。

本项目已经打包为 **Windows EXE 程序**，下载后可直接运行。

---

## 项目简介

很多 Git 初学者在使用 GitHub 时容易遇到这些问题：

- 不知道应该先 `git init` 还是 `git remote add`
- 不清楚什么时候需要 `git pull`
- 推送时遇到 `fetch first`、`no configured push destination` 不知道怎么处理
- 中文文件名在 Git 输出中显示成八进制乱码
- 不知道 `.gitignore` 应该怎么写
- 想恢复旧版本文件，但记不住命令

Git Tool 5.0 的目标就是把这些操作尽量图形化、按钮化，让新手可以通过窗口完成常见 Git 操作。

---

## 主要功能

### 一键上传到 GitHub

支持通过一个按钮完成常见上传流程：

1. 选择项目文件夹
2. 自动检测是否为 Git 仓库
3. 如果不是 Git 仓库，可自动初始化
4. 自动检测是否配置远程仓库 `origin`
5. 可输入 GitHub 仓库地址
6. 选择添加全部文件或部分文件
7. 输入提交说明
8. 自动推送到 GitHub

适合第一次上传项目到 GitHub 的用户使用。

---

### 仓库管理

支持以下仓库相关操作：

- 初始化仓库
- 克隆远程仓库
- 添加远程 `origin`
- 修改远程仓库地址
- 查看远程仓库
- 查看当前分支
- 修改默认分支名称
- 创建分支
- 切换分支

---

### 提交与同步

支持常见提交和同步操作：

- 查看状态
- 添加全部文件
- 添加部分文件
- 添加提交说明
- 智能推送
- 拉取远程更新
- 检查本地与远程同步状态

其中“检查同步”可以帮助判断当前仓库是：

- 本地领先远程
- 本地落后远程
- 本地和远程分叉
- 本地与远程一致

---

### 日志与版本恢复

支持查看 Git 提交历史，并提供版本恢复相关功能：

- 查看日志
- 查看提交 ID
- 切换到指定版本
- 恢复指定文件到旧版本
- 在日志中点击提交 ID 进行版本切换操作

适合在代码写坏后，快速找回之前的版本。

---

### `.gitignore` 忽略文件管理

内置多种 `.gitignore` 模板：

- UE5 / Unreal Engine 5
- Unity
- Python
- Node.js / 前端项目
- C / C++ / Visual Studio
- Java / Maven / Gradle
- Go
- 通用 Web 项目
- 通用模板

支持：

- 自动生成 `.gitignore`
- 打开并修改已有 `.gitignore`
- 添加自定义忽略模板
- 修改已有模板
- 打开模板文件夹

在精简模式下，如果当前仓库已经存在 `.gitignore`，会优先显示“修改忽略文件”按钮。

---

### 中文显示与换行符处理

针对 Windows 用户常见问题，提供辅助修复功能：

- 修复 Git 中文文件名显示为八进制乱码的问题
- 修复 LF / CRLF 换行符警告
- 对 warning / error 输出进行高亮提示
- 对部分常见 Git 报错给出中文说明

---

### 界面与布局自定义

支持按钮布局自定义：

- 拖拽按钮排序
- 隐藏不常用按钮
- 显示隐藏按钮
- 设置每行按钮数量
- 添加自定义按钮
- 导入 / 导出自定义按钮
- 导入 / 导出布局文件
- 重置布局

适合根据自己的使用习惯调整界面。

## 推荐使用流程

### 第一次上传项目到 GitHub

1. 在 GitHub 新建一个空仓库
2. 打开 Git Tool
3. 点击“选择仓库”，选择本地项目文件夹
4. 点击“一键上传到 GitHub”
5. 如果当前文件夹不是 Git 仓库，选择自动初始化
6. 输入 GitHub 仓库地址，例如：

```bash
https://github.com/用户名/仓库名.git
```

7. 选择添加全部文件或部分文件
8. 输入提交说明
9. 等待推送完成

---

### 日常更新项目

1. 打开 Git Tool
2. 选择项目仓库
3. 点击“查看状态”
4. 点击“一键上传到 GitHub”
5. 输入本次修改说明
6. 推送到 GitHub

---

### 从 GitHub 拉取更新

如果远程仓库有别人提交的内容，或者你在网页端修改过文件，可以点击：

```text
拉取
```

然后再继续提交和推送。

---

### 恢复旧版本文件

1. 点击“查看日志”
2. 找到想恢复到的提交 ID
3. 点击“恢复指定文件到旧版本”
4. 输入提交 ID 和文件路径
5. 确认恢复后重新提交

## 常见问题

### 1. 为什么打开 EXE 后仍然提示找不到 Git？

本工具只是 Git 的图形化辅助工具，本身不包含 Git。  
你需要先安装 Git for Windows。

安装完成后，重新打开 Git Tool。

---

### 2. 推送时报 `fetch first` 是什么原因？

说明 GitHub 远程仓库里有本地没有的提交。  
通常需要先点击“拉取”，把远程内容同步到本地，然后再推送。

---

### 3. 为什么 Git 输出中文文件名变成了数字和反斜杠？

这是 Git 的中文路径显示设置问题。  
可以点击工具中的“修复中文显示”按钮进行修复。

---

### 4. LF will be replaced by CRLF 是错误吗？

这通常不是严重错误，而是 Windows 换行符提示。  
如果不想看到该提示，可以使用工具中的“修复换行符警告”。

---

### 5. `.gitignore` 是干什么的？

`.gitignore` 用来告诉 Git 哪些文件不需要提交。  
例如 UE5、Unity、Python 项目中会产生很多缓存、临时文件和编译文件，这些通常不应该上传到 GitHub。

---

### 6. 已经生成 `.gitignore` 后还能修改吗？

可以。  
如果当前仓库已经存在 `.gitignore`，工具会显示“修改忽略文件”按钮，可以直接打开并编辑。

---

### 7. 自定义按钮有什么用？

自定义按钮可以保存你常用的 Git 命令，例如：

```bash
status -sb
log --oneline -n 5
branch -a
```

添加后就可以像普通按钮一样点击执行。
